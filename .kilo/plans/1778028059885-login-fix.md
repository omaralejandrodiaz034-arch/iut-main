## 🔧 FLUJO DE AUTENTICACIÓN

### Para usuarios existentes:
1. Ingresan cédula y contraseña
2. Sistema verifica hash de contraseña
3. Autentica y redirige según rol:
   - Admin → `usuarios.index`
   - Usuario normal → `bienes.index`

### Para nuevos usuarios (primer ingreso):
1. Ingresan cédula, dejan contraseña en blanco
2. Sistema busca en API externa / JSON local
3. Si existe, redirige a `auth.set_password.form`
4. Usuario crea contraseña
5. Se crea cuenta local y autentica automáticamente

## ⚠️ NOTAS IMPORTANTES

1. **Middleware `prevent-back`**: La ruta `/login` tiene `middleware('prevent-back')` en la ruta raíz, pero las rutas de auth usan `middleware('guest')` que previene acceso si ya está autenticado.

2. **Redirección post-login**: Configurada en `AuthController@login` según rol del usuario.

3. **Logout**: Disponible en `POST /logout` para usuarios autenticados.

4. **Session handling**: El sistema usa sesiones de Laravel con regeneración de ID tras login.
