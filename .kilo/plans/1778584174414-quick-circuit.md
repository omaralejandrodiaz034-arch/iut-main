# Base de Datos Actual - Sistema de Gestión de Inventario de Bienes (IUT)

## Descripción General
Este documento describe la estructura actual de la base de datos del Sistema de Gestión de Inventario de Bienes de la IUT, basada en las migraciones de Laravel existentes.

## Tablas Principales

### 1. organismos
- **id** (PK, bigint)
- **codigo** (varchar(50), único)
- **nombre** (varchar(150))
- **created_at**, **updated_at** (timestamps)

### 2. unidades_administradoras
- **id** (PK, bigint)
- **organismo_id** (FK → organismos.id)
- **codigo** (varchar(50))
- **nombre** (varchar(150))
- **created_at**, **updated_at** (timestamps)
- Índices: idx_ua_organismo (organismo_id)
- Único: uq_ua_org_codigo (organismo_id, codigo)

### 3. dependencias
- **id** (PK, bigint)
- **unidad_administradora_id** (FK → unidades_administradoras.id)
- **responsable_id** (FK → responsables.id, nullable)
- **codigo** (varchar(50))
- **nombre** (varchar(150))
- **created_at**, **updated_at** (timestamps)
- Índices: idx_dep_ua (unidad_administradora_id), idx_dep_responsable (responsable_id)
- Único: uq_dep_ua_codigo (unidad_administradora_id, codigo)

### 4. tipos_responsables
- **id** (PK, bigint)
- **nombre** (varchar(80), único)

### 5. responsables
- **id** (PK, bigint)
- **tipo_id** (FK → tipos_responsables.id)
- **cedula** (varchar(20), único)
- **nombre** (varchar(150))
- **correo** (varchar(150), nullable)
- **telefono** (varchar(50), nullable)
- Índice: idx_responsable_tipo (tipo_id)

### 6. bienes
- **id** (PK, bigint)
- **dependencia_id** (FK → dependencias.id)
- **codigo** (varchar(8), único)
- **descripcion** (varchar(255))
- **precio** (decimal(15,2), default 0)
- **fotografia** (varchar(255), nullable)
- **estado** (varchar(50), default 'ACTIVO')
- **fecha_registro** (date, nullable)
- **tipo_bien** (enum: ELECTRONICO, MUEBLE, VEHICULO, OTRO)
- **caracteristicas** (json/array)
- **created_at**, **updated_at** (timestamps)
- Índices: Índice estándar en dependencia_id (FK)
- Índices adicionales agregados en migración 2026_05_06_003000

### 7. bienes_electronicos
- **id** (PK, bigint)
- **bien_id** (FK → bienes.id, único)
- **subtipo** (varchar(20), nullable) // CPU, PANTALLA, ROUTER
- **procesador** (varchar(255), nullable)
- **memoria** (varchar(255), nullable)
- **almacenamiento** (varchar(255), nullable)
- **pantalla** (varchar(255), nullable)
- **serial** (varchar(255), nullable)
- **garantia** (date, nullable)
- **created_at**, **updated_at** (timestamps)

### 8. bienes_mobiliarios
- **id** (PK, bigint)
- **bien_id** (FK → bienes.id, único)
- **material** (varchar(255), nullable)
- **dimensiones** (varchar(255), nullable)
- **color** (varchar(100), nullable)
- **capacidad** (varchar(100), nullable)
- **cantidad_piezas** (integer, nullable)
- **acabado** (varchar(100), nullable)
- **created_at**, **updated_at** (timestamps)

### 9. bienes_vehiculos
- **id** (PK, bigint)
- **bien_id** (FK → bienes.id, único)
- **placa** (varchar(20), nullable)
- **chasis** (varchar(50), nullable)
- **motor** (varchar(50), nullable)
- **tipo_combustible** (varchar(50), nullable)
- **color** (varchar(50), nullable)
- **kilometraje** (integer, nullable)
- **created_at**, **updated_at** (timestamps)

### 10. bienes_otros
- **id** (PK, bigint)
- **bien_id** (FK → bienes.id, único)
- **especificaciones** (text, nullable)
- **created_at**, **updated_at** (timestamps)

### 11. bienes_desincorporados
- **id** (PK, bigint)
- **bien_id** (FK → bienes.id, único)
- **fecha_desincorporacion** (date)
- **motivo** (text)
- **valor_desincorporacion** (decimal(15,2))
- **documento_soporte** (varchar(255), nullable)
- **created_at**, **updated_at** (timestamps)

### 12. movimientos
- **id** (PK, bigint)
- **bien_id** (FK → bienes.id, nullable)
- **subject_type** (varchar(255), nullable) // Para polimorfismo
- **subject_id** (bigint, nullable)
- **tipo** (varchar(80))
- **fecha** (timestamp, default CURRENT_TIMESTAMP)
- **observaciones** (text, nullable)
- **usuario_id** (FK → usuarios.id, nullable)
- Índices: idx_mov_bien (bien_id), idx_mov_fecha (fecha), idx_mov_subject (subject_type, subject_id)

### 13. historial_movimientos
- **id** (PK, bigint)
- **movimiento_id** (FK → movimientos.id)
- **fecha** (timestamp, default CURRENT_TIMESTAMP)
- **detalle** (text)
- Índice: idx_hist_mov (movimiento_id)

### 14. usuarios
- **id** (PK, bigint)
- **rol_id** (FK → roles.id)
- **cedula** (varchar(20), único)
- **nombre** (varchar(150))
- **correo** (varchar(150), único)
- **hash_password** (varchar(255), nullable)
- **activo** (boolean, default true)
- **created_at**, **updated_at** (timestamps)
- Índice: idx_usuario_rol (rol_id)

### 15. roles
- **id** (PK, bigint)
- **nombre** (varchar(80), único)

### 16. auditoria
- **id** (PK, bigint)
- **tipo** (varchar(50))
- **descripcion** (text)
- **usuario_id** (FK → usuarios.id, nullable)
- **ip_address** (varchar(45), nullable)
- **user_agent** (text, nullable)
- **created_at** (timestamp)

### 17. reportes
- **id** (PK, bigint)
- **titulo** (varchar(255))
- **descripcion** (text)
- **tipo** (varchar(50))
- **formato** (varchar(20))
- **parametros** (json)
- **usuario_id** (FK → usuarios.id)
- **fecha_generacion** (timestamp)
- **created_at**, **updated_at** (timestamps)

### 18. eliminados
- **id** (PK, bigint)
- **tipo** (varchar(50))
- **datos** (json)
- **fecha_eliminacion** (timestamp)
- **created_at**, **updated_at** (timestamps)

## Relaciones Clave

### Jerarquía Organizacional
- organismo → (1:N) unidad_administradora
- unidad_administradora → (1:N) dependencia
- dependencia → (1:N) bien (a través de dependencia_id en bienes)
- dependiente → (1:1) responsable (opcional, a través de responsable_id)

### Bienes y Sus Detalles
- bien → (1:1) bien_electronico (opcional)
- bien → (1:1) bien_mobiliario (opcional)
- bien → (1:1) bien_vehiculo (opcional)
- bien → (1:1) bien_otro (opcional)
- bien → (1:1) bien_desincorporado (opcional)
- bien → (1:N) movimiento (opcional)
- bien → (1:N) auditoria (opcional, a través del trait AuditableTrait)

### Movimiento
- movimiento → (1:1) historial_movimiento
- movimiento → (N:1) bien (opcional)
- movimiento → (N:1) usuario (opcional)
- movimiento → (N:1) sujeto polimórfico (opcional)

### Usuarios y Roles
- usuario → (N:1) rol
- usuario → (1:N) movimiento (como usuario que realiza el movimiento)
- usuario → (1:N) reporte (como usuario que genera el reporte)
- usuario → (1:N) auditoria (opcional, a través del trait AuditableTrait)

## Notas Adicionales
1. La tabla `bienes` utiliza columnas enum para `estado` y `tipo_bien` (PHP 8.1+ Enums mapeados a strings en BD).
2. La columna `caracteristicas` en `bienes` almacena atributos adicionales en formato JSON/array.
3. Las tablas de detalles de bienes (`bienes_electronicos`, `bienes_mobiliarios`, etc.) tienen relación 1:1 con `bienes` y utilizan `bien_id` como clave primaria y foránea.
4. El sistema utiliza soft deletes implícitos mediante la tabla `eliminados` y el modelo `BienDesincorporado` para bienes retirados.
5. Las migraciones muestran evolución histórica con ajustes recientes (2026) para índices y estructura.

## Tecnologías y Características
- Motor: SQLite (desarrollo) / Compatible con MySQL/MariaDB/PostgreSQL
- ORM: Eloquent (Laravel 12)
- Patrones: Herencia mediante tablas relacionadas 1:1 para tipos de bien
- Auditoría: Trait `AuditableTrait` implementado en modelo `Bien`
- Índices: Estratégicos para búsquedas comunes por código, nombre, fechas
- Integridad: Claves foráneas con cascada apropiada
- Extensibilidad: Polimorfismo en movimientos para sujetos diversos

Este esquema representa el estado actual de la base de datos al 12 de mayo de 2026, basado en las migraciones disponibles en el repositorio.
