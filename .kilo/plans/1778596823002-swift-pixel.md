# Plan: Fix Report Generation in BienController

## Problem Analysis

The `generarReporte` method in `BienController.php` has critical issues preventing report generation:

### Issues Identified

1. **Missing `determinarTipoReporte()` method** (line 579)
   - Called but not defined in the class
   - Causes fatal error when report generation is attempted

2. **Type mismatch between validation and generation**
   - `esTipoReporteValido()` expects: `['pdf', 'excel', 'csv', 'html']`
   - `generarReportePorTipo()` uses: `['dependencia', 'unidad', 'organismo', 'tipo_bien', 'estado', 'fecha', 'default']`
   - These don't align, so even with a valid type, it would fail validation

3. **The report type determination logic is incomplete**
   - Should determine report type based on which filters are applied
   - Currently no logic to map filters to report types

## Solution Plan

### Fix 1: Add the missing `determinarTipoReporte()` method

Add a private method that determines the report type based on applied filters:

```php
private function determinarTipoReporte(array $filtros): string
{
    // Priority: more specific filters first
    if (!empty($filtros['dependencias'])) {
        return 'dependencia';
    }
    if (!empty($filtros['unidad_id'])) {
        return 'unidad';
    }
    if (!empty($filtros['organismo_id'])) {
        return 'organismo';
    }
    if (!empty($filtros['tipo_bien'])) {
        return 'tipo_bien';
    }
    if (!empty($filtros['estado'])) {
        return 'estado';
    }
    if (!empty($filtros['fecha_desde']) || !empty($filtros['fecha_hasta'])) {
        return 'fecha';
    }
    return 'default';
}
```

### Fix 2: Update `esTipoReporteValido()` to accept report types

Change the validation method to accept the actual report types used:

```php
private function esTipoReporteValido($tipoReporte): bool
{
    $tiposValidos = ['default', 'dependencia', 'unidad', 'organismo', 'tipo_bien', 'estado', 'fecha'];
    return in_array($tipoReporte, $tiposValidos);
}
```

## Implementation Steps

1. Add `determinarTipoReporte()` private method to `BienController`
2. Update `esTipoReporteValido()` to accept the correct report type values
3. Test the fix by accessing the report URL with filters applied
