# Plan: Fix Blank "Crear Dependencia" Page

## Problem
The `create` method in `DependenciaController` is empty (lines 49-52), returning nothing. This causes a blank page when accessing `resources/views/dependencias/create.blade.php`. The view requires these variables:
- `$unidades` (collection of UnidadAdministradora)
- `$responsables` (collection of Responsable)
- `$proximoCodigo` (string, next 8-digit code suggestion)
- `$sugerenciasPorUnidad` (array mapping unidad_id → next code)

Additionally, the `store` method (lines 44-47) is also empty, so form submission would fail.

## Solution
Implement both methods in `app/Http/Controllers/DependenciaController.php`:

### 1. Add imports
At the top of the file, add:
```php
use Illuminate\Support\Facades\DB;
use Illuminate\Validation\Rule;
```

### 2. Implement `create(Request $request)`
```php
public function create(Request $request)
{
    $unidades = UnidadAdministradora::all();
    $responsables = Responsable::all();

    // Calcular el próximo código global (máximo existente + 1)
    $maxCodigoGlobal = Dependencia::whereRaw("codigo REGEXP '^[0-9]+$'")
        ->max(DB::raw('CAST(codigo AS UNSIGNED)'));

    // Para cada unidad, calcular su próximo código based on its own max
    $sugerenciasPorUnidad = [];
    foreach ($unidades as $unidad) {
        $maxCodigoUnidad = Dependencia::where('unidad_administradora_id', $unidad->id)
            ->whereRaw("codigo REGEXP '^[0-9]+$'")
            ->max(DB::raw('CAST(codigo AS UNSIGNED)'));

        $siguiente = $maxCodigoUnidad ? $maxCodigoUnidad + 1 : 1;
        $sugerenciasPorUnidad[$unidad->id] = str_pad((string)$siguiente, 8, '0', STR_PAD_LEFT);
    }

    // Determinar el código inicial a mostrar
    $proximoCodigo = $sugerenciasPorUnidad[$unidades->first()?->id] ?? '00000001';

    return view('dependencias.create', compact('unidades', 'responsables', 'proximoCodigo', 'sugerenciasPorUnidad'));
}
```

### 3. Implement `store(Request $request)`
```php
public function store(Request $request)
{
    $validated = $request->validate([
        'unidad_administradora_id' => 'required|exists:unidades_administradoras,id',
        'codigo' => [
            'required',
            'string',
            'max:8',
            'regex:/^[0-9]+$/',
            Rule::unique('dependencias')
                ->where(function ($query) use ($request) {
                    return $query->where('unidad_administradora_id', $request->unidad_administradora_id);
                })
        ],
        'nombre' => 'required|string|max:40',
        'responsable_id' => 'nullable|exists:responsables,id',
    ]);

    Dependencia::create($validated);

    return redirect()
        ->route('dependencias.index')
        ->with('success', '✅ Dependencia registrada exitosamente.');
}
```

## Notes
- The code uses `whereRaw` with `REGEXP` to ensure only numeric codes are considered for max calculations (matching pattern from `UnidadAdministradoraController`).
- The composite unique constraint (`unidad_administradora_id`, `codigo`) is enforced at DB level and validated via `Rule::unique()->where()`.
- The `store` method uses simple validation and creation; error handling will automatically redirect back with errors if validation fails.
- The `proximoCodigo` default is taken from the first unidad's suggestion. If no unidades exist, falls back to '00000001' (though the select would be empty).
